What is a pre-processor? 

If you haven't read the notes on C compilation process, then here is a reminder. It removed comments, copies all the code 
in the header files you included at the top of your C program into your source code and fullfills macros, or otherwise known as 
pre-processor directives, which is just a fancy word for "orders".

Macros in C are defined using the "#" symbol, or they can be built-in ones as well, like __FILE__ which represents the name of the file
where it is used.

There are a lot of built-in macros in C, and I mean a lot. And truth be told you don't need to know all of them, even knowing 10 or so would be enough.

But you can define macros on your own too.

Like so,

#define PI 3.1416251

here we are defining a constant called PI, now if you use the word "PI" anywhere in your code, it will be replaced
by the pre-processor with the value we assign to it.

Like say you define a function like so:

float area(float radius){             After Pre-processor    float area(float radius){
    return PI * pow(radius, 2);      -------------------->      return 3.1416251 * pow(radius, 2);
}                                                            }

The above define function uses PI to calculate the area of a circle, after you pass your code to the 
pre-processor the function will look like shown above.

This is what I meant when I said the pre-processor is responsible for text-subsitution for fullfilling macros in a C program.

The words "macro" in iteself just means something large or broad, and it makes sense because when you define a macro in C 
it is replaced by its value everywhere in your program.


Using Macros in other ways.


Since all the #define command does, is tell the pre-processor to replace a word with the value you provide after.
You can use this to your advantage for making some pretty generic functions.
