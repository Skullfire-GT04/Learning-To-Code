Functions are the most basic building block of any application or program, so it is only fair that they get a good 
explanation, and this explanation doesn't just apply to functions in C but just functions in programming in general.

A function is just a block of code which can be executed when the function is called. They were introduced in programming because
it didn't make sense to write the same code in different places again and again, it was both in-efficient and primitive to do so.


Now functions in C and most other programming languages are supposed to be unique. But in most modern programming languages, there is a concept
of function overloading. But since C doesn't allow function overloading, it is superflous to learn about it, but meh here you go.

Function overloading is when you define multiple functions with the same name which perform different things.
It is possible to define many functions with the same name because as long as the function signature is unique the language will allow for the name to be the same.

Function Signature: The combination of the return type, the name and the parameters of a function constitues the function signature, it is a way of uniquely
                    identifying functions for the compiler. In C even if the function signature is different, two functions cannot share the same name.
                    In other languages, if the compiler/interpreter allows it, you can overload functions (see below).


Function Definition/Implementation: It is the actual code which the function represents when it is called. Like the actual logic which is implemented behind every
                                    function call.


Function Overloading: When you write two seperate functions with different signatures but the same name, it is called function overloading. It is a technique
                      used to make generic function names for statically types languages like C++, Java, Rust and so on. 


Generic Functions: When working with statically typed languages, it is really hard to define an algorithm which works for all types of data, because the 
                   functions only take in a particular type of data as input arguments. And that results into programmers needing to create different functions
                   with different names for the same purpose essentially. And that's where the need for generic functions come in. The word generic means 'something 
                   in general, or average', and that's what these functions are. They can take any type of input arguments, hence eliminating the need for 
                   different functions.

                   Like, in python a genric function would look like:

                   def add(a, b): 
                        return a + b

                   here a, b can be integers, string, float or anything. But if we were to define the same function in C or any other statically typed language
                   we would need to define a sperate function for each data type.
